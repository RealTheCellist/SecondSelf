# Second Self 와이어프레임용 화면 명세

## 문서 목적

이 문서는 `Second Self` 7일 체크인 실험용 MVP를 와이어프레임으로 옮기기 위한 화면 명세서다.

디자인 목표는 화려한 기능 설명이 아니라, 다음 흐름이 자연스럽게 이어지도록 만드는 것이다.

1. 질문을 고른다
2. 짧게 입력한다
3. 해석 1줄을 본다
4. 체크인이 쌓이면 패턴을 본다

## 화면 구조 개요

- 화면 1: 첫 질문 선택
- 화면 2: 체크인 작성
- 화면 3: 결과 및 패턴 피드

초기 실험에서는 이 3개 화면만 정의한다.

## 공통 UX 원칙

- 한 화면에 한 가지 행동만 유도한다
- 첫 세션에서 스크롤 길이는 짧아야 한다
- 버튼 라벨은 의미가 즉시 이해되어야 한다
- 설명 텍스트보다 입력과 결과가 먼저 보여야 한다

## 화면 1. 첫 질문 선택

### 목적

- 사용자가 제품 설명을 읽지 않고도 바로 시작하게 만든다

### 핵심 행동

- 질문 3개 중 1개 선택
- 시작 버튼 탭

### 레이아웃

1. 상단 여백
2. 헤드라인
3. 짧은 서브카피
4. 질문 카드 3개
5. 하단 고정 CTA 버튼

### 컴포넌트

- 헤드라인 텍스트
- 서브카피 텍스트
- 선택형 질문 카드 3개
- CTA 버튼 1개

### 상태

- 기본 상태: 아무 질문도 선택되지 않음
- 선택 상태: 카드 1개 활성화
- 비활성 CTA 상태: 질문 미선택 시 버튼 비활성
- 활성 CTA 상태: 질문 선택 시 버튼 활성

### 인터랙션

- 질문 카드를 탭하면 해당 카드가 선택 상태가 된다
- 다른 카드를 탭하면 기존 선택은 해제되고 새 카드가 선택된다
- CTA 탭 시 체크인 작성 화면으로 이동한다

### 카피 참조

- `SECOND_SELF_SCREEN_COPY.md`의 `화면 1. 첫 질문 선택 화면` 사용

## 화면 2. 체크인 작성

### 목적

- 사용자가 부담 없이 30초 안에 체크인을 완료하게 만든다

### 핵심 행동

- 자유 텍스트 입력
- 감정 태그 1~2개 선택
- 완료 버튼 탭

### 레이아웃

1. 상단 진행 맥락 라벨
2. 선택한 질문 노출
3. 자유 텍스트 입력 영역
4. 감정 태그 영역
5. 하단 CTA 버튼

### 컴포넌트

- 상단 라벨
- 선택 질문 요약 텍스트
- 멀티라인 입력 필드
- 감정 태그 칩 8~12개
- CTA 버튼

### 상태

- 초기 상태: 빈 입력, 버튼 비활성
- 텍스트 입력 상태: 버튼 활성 조건 일부 충족
- 태그 선택 상태: 버튼 활성 조건 일부 충족
- 제출 가능 상태: 텍스트 + 태그 선택 완료
- 제출 로딩 상태

### 인터랙션

- 텍스트 필드는 2~4줄 높이의 멀티라인 입력으로 시작한다
- 감정 태그는 최대 2개까지 선택 가능하다
- 완료 버튼 탭 시 결과 화면으로 이동한다

### 유효성 기준

- 텍스트 최소 1자 이상
- 감정 태그 1개 이상

### 예외 상태

- 텍스트 없음: 입력 요청 문구 표시
- 태그 없음: 태그 선택 요청 문구 표시
- 저장 실패: 재시도 토스트 또는 인라인 오류 표시

### 카피 참조

- `SECOND_SELF_SCREEN_COPY.md`의 `화면 2. 체크인 작성 화면` 사용

## 화면 3. 결과 및 패턴 피드

### 목적

- 사용자가 입력 직후 `정리되어 돌아오는 느낌`을 받게 만든다
- 체크인이 누적되면 최근 패턴까지 자연스럽게 확인하게 만든다

### 핵심 행동

- 해석 1줄 읽기
- 패턴 카드 확인
- 다음 체크인 의도 형성

### 레이아웃

1. 상단 결과 헤드라인
2. 오늘의 해석 1줄
3. 오늘의 키워드 2개
4. 패턴 카드 영역
5. 하단 CTA

### 컴포넌트

- 결과 헤드라인
- 해석 1줄 텍스트 블록
- 키워드 배지 2개
- 패턴 카드 3개
- CTA 버튼

### 상태 A. 첫 체크인 직후

- 해석 1줄 노출
- 키워드 2개 노출
- 패턴 카드 영역은 빈 상태 또는 프리뷰 상태

### 상태 B. 데이터 누적 후

- 해석 1줄 노출
- 키워드 2개 노출
- 패턴 카드 3개 전체 노출

### 패턴 카드 구성

카드 1:

- 제목: 자주 나온 주제
- 내용: 상위 2~3개 주제 단어

카드 2:

- 제목: 반복 감정
- 내용: 상위 1~2개 감정 태그

카드 3:

- 제목: 최근의 나
- 내용: 규칙형 템플릿 기반 한 줄 요약

### 인터랙션

- CTA 탭 시 다음 체크인 진입 또는 홈 복귀
- 카드 자체는 초기 버전에서 상세 페이지로 연결하지 않는다

### 빈 상태 처리

- 최근 5개 미만 데이터일 때는 패턴 카드 대신 빈 상태 문구를 먼저 보여준다
- 3회 이상 누적되면 카드 일부 프리뷰를 고려할 수 있다

### 카피 참조

- `SECOND_SELF_SCREEN_COPY.md`의 `화면 3. 첫 결과 화면`, `화면 4. 패턴 피드 화면`, `빈 상태 카피` 사용

## 화면 간 전이

1. 질문 선택 완료
2. 체크인 작성
3. 결과 확인
4. 다음 날 재진입 시 질문 선택 또는 바로 체크인 시작

초기 실험에서는 분기보다 직선형 흐름을 우선한다.

## 우선순위

### P0

- 질문 선택
- 텍스트 입력
- 감정 태그 선택
- 결과 1줄 노출

### P1

- 키워드 2개 노출
- 패턴 카드 3장
- 빈 상태 문구

### P2

- 미세 애니메이션
- 카드 상세 진입
- 히스토리 리스트

## 구현 메모

- 첫 실험은 모바일 세로 화면 기준으로 설계한다
- 입력과 결과는 한 손 사용 기준으로 CTA가 하단에 있어야 한다
- 결과 화면은 시각적으로 `분석 결과`보다 `조용한 반사`에 가까운 분위기가 적합하다
- 카드 수는 3장을 넘기지 않는 것이 좋다

## 디자이너 핸드오프 체크리스트

- 첫 질문 선택 화면에서 무엇을 해야 하는지 3초 안에 이해되는가
- 체크인 작성 화면이 30초 내 완료 가능해 보이는가
- 결과 화면의 첫 시선이 해석 1줄에 가는가
- 패턴 카드가 정보량보다 해석 경험에 기여하는가
- 전체 흐름이 `기록 앱`보다 `반사 도구`에 가깝게 느껴지는가
