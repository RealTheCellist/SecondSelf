# Second Self 사용자 테스트 스크립트

## 목적

이 문서는 `Second Self` 7일 체크인 실험용 MVP의 1차 사용자 테스트를 위한 진행 스크립트다.

목표는 사용자가 이 앱을 `기록 앱`이 아니라 `나를 비춰주는 도구`로 느끼는지 확인하는 것이다.

## 테스트 대상

- 커리어 전환기 사용자
- 삶의 방향을 재정리 중인 사용자
- 짧은 자기 점검에는 열려 있지만 긴 저널링 습관은 없는 사용자

## 테스트 시간

- 총 15~20분
- 제품 경험 5분
- 인터뷰 10~15분

## 진행 전 안내

진행자가 사용자에게 안내할 문구:

`이건 완성 제품이 아니라 실험용 프로토타입입니다. 사용법을 평가하는 게 아니라, 이 경험이 어떻게 느껴지는지 보려는 테스트예요. 헷갈리거나 이상한 점이 있으면 바로 말해주세요.`

## 테스트 시나리오 1

### 상황

`요즘 일과 진로 사이에서 방향성을 자주 고민하는 사람` 역할로 사용한다.

### 과제

1. 앱을 켜고, 가장 끌리는 질문을 하나 고른다
2. 짧게 체크인을 남긴다
3. 결과 화면을 보고 지금 느껴지는 점을 말한다

### 관찰 포인트

- 첫 화면에서 무엇을 해야 하는지 바로 이해하는가
- 체크인 작성이 부담스럽지 않은가
- 해석 1줄을 읽고 멈추는가, 그냥 지나치는가

## 테스트 시나리오 2

### 상황

`최근 감정 기복이 반복된다고 느끼는 사람` 역할로 사용한다.

### 과제

1. 감정 중심 질문을 선택한다
2. 감정 태그를 고르며 체크인을 작성한다
3. 결과 화면에서 가장 눈에 남는 부분을 말한다

### 관찰 포인트

- 감정 태그 선택이 자연스러운가
- 결과 화면에서 어떤 부분을 가장 먼저 읽는가
- 사용자가 `맞다`, `애매하다`, `과장됐다` 중 어디에 가까운 반응을 보이는가

## 테스트 시나리오 3

### 상황

`며칠간 기록이 쌓인 뒤 패턴을 확인하는 사람` 역할로 사용한다.

### 준비

- 진행자가 `샘플 채우기` 버튼으로 데이터가 쌓인 상태를 만든다

### 과제

1. 누적된 결과 화면을 본다
2. 패턴 카드 3장을 읽는다
3. 다시 들어오고 싶어지는지 말한다

### 관찰 포인트

- 패턴 카드가 실제 가치처럼 느껴지는가
- 가장 설득력 있는 카드는 무엇인가
- 다시 들어오고 싶은 이유가 생기는가

## 인터뷰 질문

### 첫 인상

- 이 앱이 처음엔 어떤 앱처럼 느껴졌나요?
- 기록 앱 같았나요, 아니면 다른 종류의 도구 같았나요?

### 사용 흐름

- 어디서 가장 자연스러웠나요?
- 어디서 가장 잠깐 멈칫했나요?

### 결과 화면

- 해석 1줄은 얼마나 맞다고 느껴졌나요?
- 패턴 카드가 도움이 됐나요, 아니면 아직 약하게 느껴졌나요?

### 재방문 의도

- 내일 다시 들어올 이유가 생겼나요?
- 들어온다면 어떤 순간에 다시 열 것 같나요?

### 표현 검증

- `기록을 쌓게 하는 앱`보다 `최근의 나를 비춰주는 앱`이라는 설명이 맞게 느껴지나요?

## 기록 템플릿

테스트마다 아래 형식으로 짧게 남긴다.

- 사용자 유형:
- 첫 인상:
- 가장 좋았던 부분:
- 가장 헷갈렸던 부분:
- 해석 1줄 반응:
- 패턴 카드 반응:
- 재방문 의도:
- 바로 고쳐야 할 점:

## 성공 신호

- 사용자가 결과 화면에서 잠시 멈추고 읽는다
- `맞다`, `생각해볼 만하다`, `비춰주는 느낌` 같은 표현이 나온다
- 다시 들어오는 이유를 스스로 말할 수 있다

## 위험 신호

- `그냥 기록 앱 같다`
- `결과가 너무 뻔하다`
- `내 얘기 같지 않다`
- `다시 들어올 이유는 잘 모르겠다`
