# Second Self 하이파이 화면 고정본

## 문서 목적

이 문서는 `Second Self` 7일 체크인 실험용 MVP의 화면별 최종 카피와 레이아웃 결정을 고정한 문서다.

디자인, 구현, 테스트가 같은 기준을 보도록 하기 위한 확정본이다.

## 적용 범위

- 모바일 세로 화면 기준
- 첫 실험 버전 기준
- 추천 흐름 `화면 1 -> 화면 2 -> 화면 3 통합형 결과/패턴`

## 전체 흐름

1. 첫 질문 선택
2. 체크인 작성
3. 통합형 결과/패턴 피드

## 화면 1. 첫 질문 선택

### 목적

- 사용자가 제품 설명을 읽지 않고도 바로 시작하게 만든다

### 확정 카피

- 상단 브랜드 라벨: `Second Self`
- 헤드라인: `오늘의 나는 어디에 가장 오래 머물렀나요?`
- 서브카피: `한 가지 질문을 고르고, 짧게 적어보세요.`

질문 카드:

- `요즘 자주 붙잡히는 생각은 무엇인가요?`
- `최근 반복해서 느끼는 감정은 무엇인가요?`
- `지금의 나를 한 문장으로 표현하면 어떤가요?`

CTA:

- 비활성 상태: `시작하기`
- 활성 상태: `시작하기`

### 레이아웃 우선순위

1. 헤드라인
2. 질문 카드 3개
3. CTA

### 시각 메모

- 브랜드명은 작고 조용하게
- 질문 카드 선택 상태가 가장 분명해야 한다
- 첫 화면은 설명형 랜딩처럼 보이면 안 된다

## 화면 2. 체크인 작성

### 목적

- 30초 안에 입력을 끝내게 만든다

### 확정 카피

- 상단 라벨: `오늘의 체크인`
- 보조 문구: `30초면 충분합니다.`

입력 필드 플레이스홀더:

- `떠오르는 그대로 적어보세요`

감정 태그 섹션:

- 라벨: `지금 가까운 감정을 골라보세요`
- 보조 문구: `1개 또는 2개만 선택해도 충분합니다`

CTA:

- 기본: `이대로 비춰보기`
- 로딩: `비추는 중...`

오류 문구:

- 텍스트 없음: `한두 문장만 적어도 충분합니다`
- 태그 없음: `감정 태그를 1개 이상 골라주세요`
- 저장 실패: `잠시 연결이 불안정해요. 다시 한번 시도해주세요`

### 감정 태그 초안

- 불안
- 압박감
- 기대
- 자신감
- 혼란
- 안정감
- 의욕
- 무기력

### 레이아웃 우선순위

1. 선택한 질문
2. 텍스트 입력
3. 감정 태그
4. CTA

### 시각 메모

- 입력 영역이 화면의 중심이어야 한다
- 태그는 칩 형태로 짧고 가볍게
- CTA는 하단 고정

## 화면 3. 통합형 결과/패턴 피드

### 목적

- 입력 직후 해석 1줄을 먼저 보여주고
- 데이터가 쌓일수록 패턴 카드가 이어지게 만든다

### 확정 카피

- 헤드라인: `지금의 나를 비추면`

해석 1줄 예시:

- `요즘 당신은 결과보다 방향을 더 오래 붙잡고 있습니다.`

키워드 예시:

- `방향성`
- `불확실성`

보조 문구:

- `완전히 맞지 않아도 괜찮습니다. 지금의 나를 가볍게 비춰보는 중이니까요.`

패턴 섹션 헤드라인:

- `최근의 나는 이렇게 반복되고 있어요`

패턴 섹션 서브카피:

- `최근 5번의 체크인에서 자주 나온 흐름입니다`

카드 제목:

- `자주 나온 주제`
- `반복 감정`
- `최근의 나`

CTA:

- 데이터 부족 상태: `내일 다시 체크인하기`
- 데이터 누적 상태: `다음 체크인 남기기`

### 데이터 부족 상태 카피

- 빈 상태 헤드라인: `아직은 오늘의 한 줄이 더 정확합니다`
- 빈 상태 본문: `체크인이 몇 번 더 쌓이면, 반복되는 감정과 주제를 함께 보여드릴게요.`

### 카드 내용 예시

- 자주 나온 주제: `일, 방향성, 관계`
- 반복 감정: `불안, 압박감`
- 최근의 나: `요즘 당신은 성과보다 앞으로의 방향을 더 자주 고민하고 있습니다`

### 레이아웃 우선순위

1. 해석 1줄
2. 키워드 2개
3. 보조 문구
4. 패턴 섹션
5. CTA

### 시각 메모

- 첫 시선은 해석 1줄에 가야 한다
- 패턴 카드는 같은 폭, 같은 리듬 유지
- 분석 도구보다 조용한 반사 도구처럼 보여야 한다

## 상태별 노출 규칙

### 상태 A. 첫 체크인 직후

- 해석 1줄 노출
- 키워드 2개 노출
- 보조 문구 노출
- 빈 상태 박스 노출
- CTA는 `내일 다시 체크인하기`

### 상태 B. 3회 이상 누적

- 해석 1줄 노출
- 키워드 2개 노출
- 패턴 카드 프리뷰 가능
- CTA는 `다음 체크인 남기기`

### 상태 C. 5회 이상 누적

- 해석 1줄 노출
- 키워드 2개 노출
- 패턴 카드 3장 전체 노출
- CTA는 `다음 체크인 남기기`

## 인터랙션 원칙

- 화면 1에서 질문 선택 전 CTA는 비활성
- 화면 2에서 텍스트와 태그가 모두 입력되면 CTA 활성
- 화면 3의 카드들은 초기 버전에서 상세 페이지로 이동하지 않음
- CTA는 항상 다음 행동 하나만 제안

## 디자인 체크포인트

- 첫 화면이 온보딩 소개 페이지처럼 보이지 않는가
- 작성 화면이 입력 부담보다 가벼워 보이는가
- 결과 화면에서 해석 1줄이 가장 먼저 읽히는가
- 패턴 카드가 많아 보이기보다 정리되어 보이는가
