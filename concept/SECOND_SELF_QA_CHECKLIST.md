# Second Self QA 체크리스트

## 목적

이 문서는 `Second Self` 7일 체크인 실험용 MVP의 내부 QA를 빠르게 반복하기 위한 체크리스트다.

대상 버전:

- 정적 프로토타입 기준
- 파일: [index.html](C:/Users/haneu/SecondSelf/index.html:1), [styles.css](C:/Users/haneu/SecondSelf/styles.css:1), [app.js](C:/Users/haneu/SecondSelf/app.js:1)

## QA 준비

테스트 전 아래 상태를 준비한다.

1. 브라우저에서 앱을 연다
2. 필요 시 `기록 초기화` 버튼으로 상태를 비운다
3. 패턴 상태를 빠르게 보고 싶으면 `샘플 채우기` 버튼을 사용한다

## 핵심 흐름

### 질문 선택 화면

- 앱 진입 시 질문 선택 화면이 먼저 보인다
- 질문 3개가 한 화면 안에 보인다
- 질문 미선택 상태에서는 `시작하기` 버튼이 비활성이다
- 질문 선택 후 `시작하기` 버튼이 활성화된다

### 체크인 작성 화면

- 선택한 질문이 상단에 다시 보인다
- 텍스트와 태그 둘 다 없으면 CTA가 비활성이다
- 텍스트 입력만 하면 CTA는 여전히 비활성이다
- 감정 태그를 1개 이상 선택하면 CTA가 활성화된다
- 감정 태그는 최대 2개까지만 선택된다
- 3번째 감정을 누르면 제한 안내 문구가 보인다

### 제출 및 결과 화면

- 제출 후 결과 화면으로 정상 이동한다
- 해석 1줄이 결과 화면에서 가장 먼저 보인다
- 첫 체크인 직후에는 빈 상태 박스가 보인다
- 3회 이상 누적되면 패턴 카드 프리뷰가 보인다
- 5회 이상 누적되면 패턴 카드 3장이 모두 자연스럽게 보인다

## 상태별 체크

### 빈 상태

- 체크인이 하나도 없을 때 인사이트 카드가 `0회`, `아직 없음`으로 보인다
- `첫 체크인 시작하기` CTA가 보인다

### early 상태

- 체크인이 1~2회일 때 단계 라벨이 `초기`로 보인다
- 패턴 카드 대신 안내 문구가 유지된다

### preview 상태

- 체크인이 3~4회일 때 단계 라벨이 `미리보기`로 보인다
- 패턴 카드가 나타난다
- 서브카피가 `3회 이상 쌓인 체크인을 기준으로 먼저 보이는 흐름입니다`로 보인다

### full 상태

- 체크인이 5회 이상일 때 단계 라벨이 `완성`으로 보인다
- 패턴 카드 3장이 모두 정상 노출된다
- 누적 체크인 수와 대표 감정이 함께 업데이트된다

## 데이터 동작

- `기록 초기화` 클릭 시 체크인과 이벤트 로그가 모두 초기화된다
- 초기화 후 앱은 다시 빈 상태로 보인다
- `샘플 채우기` 클릭 시 5개 샘플 체크인이 들어간다
- 샘플 채우기 후 결과 화면으로 전환된다

## 시각 확인

- 모바일 폭에서도 레이아웃이 무너지지 않는다
- 하단 버튼 영역이 겹치지 않는다
- 질문 카드 선택 상태가 분명하다
- 해석 1줄이 카드보다 먼저 읽힌다

## 기본 브라우저 점검

- Chrome
- Edge
- 모바일 브라우저 폭 에뮬레이션

## 회귀 체크

수정 후 아래 네 가지만 최소 재확인한다.

1. 질문 선택 없이 진행 불가
2. 태그/텍스트 없이 제출 불가
3. 제출 후 결과 화면 이동
4. 샘플 채우기 후 full 상태 정상 노출
