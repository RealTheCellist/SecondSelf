# Second Self 구현 태스크 분해

## 문서 목적

이 문서는 `Second Self` 7일 체크인 실험용 MVP를 실제 구현으로 옮기기 위한 작업 분해 문서다.

목표는 무엇부터 만들지, 무엇을 뒤로 미룰지 분명하게 만드는 것이다.

## 릴리스 목표

- 3개 화면 흐름 완성
- 체크인 저장 가능
- 결과 해석 1줄 표시 가능
- 데이터 누적 시 패턴 카드 3장 표시 가능

## 작업 스트림

- 제품/콘텐츠
- 프론트엔드
- 백엔드 또는 데이터 레이어
- 실험/분석

## 1. 제품/콘텐츠

### P0

- 질문 3개 최종 확정
- 감정 태그 세트 최종 확정
- 해석 1줄 템플릿 톤 확정
- 빈 상태 카피 확정

### P1

- 패턴 카드 제목/본문 템플릿 다듬기
- 정성 인터뷰 질문지 작성

## 2. 프론트엔드

### P0

- 질문 선택 화면 구현
- 체크인 작성 화면 구현
- 결과 화면 구현
- 하단 CTA 컴포넌트 구현
- 입력 검증 및 오류 문구 연결

### P1

- 패턴 카드 UI 구현
- empty/full 상태 분기
- 분석 이벤트 연결

### P2

- preview 상태 구현
- 미세 인터랙션 추가

## 3. 백엔드 또는 데이터 레이어

### P0

- 체크인 저장 스키마 정의
- 체크인 생성 API 또는 로컬 저장 함수 구현
- 최근 N개 체크인 조회 구현
- 요약 데이터 생성 함수 구현

### P1

- 키워드 추출 규칙 보강
- 패턴 카드 템플릿 규칙 보강

## 4. 실험/분석

### P0

- 이벤트 스키마 정의
- 첫 체크인 완료율 측정
- 결과 화면 도달율 측정
- 7일 내 반복 체크인 수 측정

### P1

- 정성 인터뷰 폼 작성
- 회고용 대시보드 또는 요약 시트 준비

## 추천 구현 순서

1. 카피와 질문/태그 세트 확정
2. 화면 1 질문 선택 구현
3. 화면 2 체크인 작성 구현
4. 체크인 저장 구현
5. 화면 3 해석 1줄 구현
6. 패턴 카드 집계 로직 구현
7. empty/full 상태 연결
8. 분석 이벤트 연결

## 하루 단위 스프린트 예시

### Day 1

- 질문/태그/카피 확정
- 화면 1, 화면 2 UI 뼈대 구현

### Day 2

- 입력 검증
- 저장 로직 연결
- 결과 화면 1차 구현

### Day 3

- 규칙형 요약 로직 구현
- 패턴 카드 연결

### Day 4

- 상태 분기 정리
- 분석 이벤트 연결
- QA

## QA 체크리스트

- 질문 선택 없이 다음으로 못 가는가
- 텍스트 없이 제출되지 않는가
- 태그 없이 제출되지 않는가
- 제출 후 결과 화면으로 정상 이동하는가
- 첫 체크인 직후 빈 상태 문구가 자연스러운가
- 5개 누적 후 패턴 카드가 정상 노출되는가
- 전체 흐름이 1분 내 완료 가능해 보이는가
