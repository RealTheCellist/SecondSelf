# Second Self 로우파이 와이어프레임 텍스트 스케치

## 문서 목적

이 문서는 `Second Self` 7일 체크인 실험용 MVP의 모바일 기준 로우파이 와이어프레임 텍스트 스케치다.

목표는 화면의 미감보다 다음을 빠르게 공유하는 것이다.

- 각 화면에서 무엇이 가장 먼저 보여야 하는가
- 사용자가 어떤 순서로 행동하는가
- 어떤 정보가 위에 오고, 어떤 정보가 아래로 밀리는가

## 기본 가정

- 모바일 세로 화면 기준
- 한 손 사용 기준
- 하단 CTA 고정
- 첫 세션은 짧고 직선적인 흐름

## 전체 흐름

1. 첫 질문 선택
2. 체크인 작성
3. 결과 확인
4. 데이터 누적 후 패턴 피드 확인

---

## 화면 1. 첫 질문 선택

```
+--------------------------------------------------+
|                                                  |
|                  Second Self                     |
|                                                  |
|   오늘의 나는 어디에 가장 오래 머물렀나요?       |
|   한 가지 질문을 고르고, 짧게 적어보세요.        |
|                                                  |
|   +------------------------------------------+   |
|   | ○ 요즘 자주 붙잡히는 생각은 무엇인가요? |   |
|   +------------------------------------------+   |
|                                                  |
|   +------------------------------------------+   |
|   | ○ 최근 반복해서 느끼는 감정은 무엇인가요?|   |
|   +------------------------------------------+   |
|                                                  |
|   +------------------------------------------+   |
|   | ○ 지금의 나를 한 문장으로 표현하면?      |   |
|   +------------------------------------------+   |
|                                                  |
|                                                  |
|   [ 시작하기 ]                                   |
+--------------------------------------------------+
```

### 메모

- 브랜드명은 작게, 질문 선택이 크게 보여야 한다
- 질문 카드 3개는 한 화면 안에 모두 보여야 한다
- CTA는 질문 선택 전까지 비활성 상태

### 선택 상태 예시

```
+----------------------------------------------+
| ● 요즘 자주 붙잡히는 생각은 무엇인가요?      |
+----------------------------------------------+
```

## 화면 2. 체크인 작성

```
+--------------------------------------------------+
| <                                               |
|                                                  |
|   오늘의 체크인                                  |
|   30초면 충분합니다.                             |
|                                                  |
|   요즘 자주 붙잡히는 생각은 무엇인가요?          |
|                                                  |
|   +------------------------------------------+   |
|   | 떠오르는 그대로 적어보세요                |   |
|   |                                          |   |
|   | 오늘은 자꾸 방향성 얘기로 생각이 돌아와요|   |
|   | 지금 잘 가고 있는 건지 계속 확인하게 돼요|   |
|   |                                          |   |
|   +------------------------------------------+   |
|                                                  |
|   지금 가까운 감정을 골라보세요                 |
|   1개 또는 2개만 선택해도 충분합니다            |
|                                                  |
|   [불안] [압박감] [기대] [혼란]                 |
|   [안정감] [의욕] [무기력] [자신감]             |
|                                                  |
|                                                  |
|   [ 이대로 비춰보기 ]                            |
+--------------------------------------------------+
```

### 메모

- 상단에는 진행 맥락과 선택한 질문만 남긴다
- 텍스트 입력 영역이 화면의 중심이다
- 태그는 칩 형태로 짧게 노출하고 최대 2개 선택
- CTA는 입력과 태그가 둘 다 채워져야 활성화

### 오류 상태 예시

```
텍스트 없음:
"한두 문장만 적어도 충분합니다"

태그 없음:
"감정 태그를 1개 이상 골라주세요"
```

## 화면 3. 첫 결과 화면

```
+--------------------------------------------------+
|                                                  |
|   지금의 나를 비추면                             |
|                                                  |
|   요즘 당신은 결과보다 방향을 더 오래           |
|   붙잡고 있습니다.                               |
|                                                  |
|   [방향성]   [불확실성]                          |
|                                                  |
|   완전히 맞지 않아도 괜찮습니다.                |
|   지금의 나를 가볍게 비춰보는 중이니까요.       |
|                                                  |
|   +------------------------------------------+   |
|   | 아직은 오늘의 한 줄이 더 정확합니다       |   |
|   | 체크인이 몇 번 더 쌓이면, 반복되는 감정과 |   |
|   | 주제를 함께 보여드릴게요.                 |   |
|   +------------------------------------------+   |
|                                                  |
|                                                  |
|   [ 내일 다시 체크인하기 ]                       |
+--------------------------------------------------+
```

### 메모

- 첫 시선은 반드시 해석 1줄에 꽂혀야 한다
- 키워드는 해석을 보조하는 정도로만 사용한다
- 데이터 부족 상태에서는 패턴 카드 대신 빈 상태 박스를 보여준다

---

## 화면 4. 패턴 피드 화면

체크인이 5개 정도 누적된 이후 기준.

```
+--------------------------------------------------+
|                                                  |
|   최근의 나는 이렇게 반복되고 있어요             |
|   최근 5번의 체크인에서 자주 나온 흐름입니다     |
|                                                  |
|   +------------------------------------------+   |
|   | 자주 나온 주제                           |   |
|   | 일, 방향성, 관계                         |   |
|   +------------------------------------------+   |
|                                                  |
|   +------------------------------------------+   |
|   | 반복 감정                               |   |
|   | 불안, 압박감                            |   |
|   +------------------------------------------+   |
|                                                  |
|   +------------------------------------------+   |
|   | 최근의 나                               |   |
|   | 요즘 당신은 성과보다 앞으로의 방향을    |   |
|   | 더 자주 고민하고 있습니다               |   |
|   +------------------------------------------+   |
|                                                  |
|   [ 다음 체크인 남기기 ]                         |
+--------------------------------------------------+
```

### 메모

- 카드 3장은 같은 폭, 같은 리듬으로 정렬
- 세 번째 카드가 가장 해석적이어야 한다
- 카드별 상세 진입은 초기 버전에서 없음

---

## 화면 5. 결과 화면의 누적 상태

첫 결과 화면과 패턴 피드 화면이 완전히 분리되지 않고, 한 화면 안에서 이어지는 버전도 가능하다.

```
+--------------------------------------------------+
|                                                  |
|   지금의 나를 비추면                             |
|   요즘 당신은 결과보다 방향을 더 오래           |
|   붙잡고 있습니다.                               |
|                                                  |
|   [방향성]   [불확실성]                          |
|                                                  |
|   최근의 나는 이렇게 반복되고 있어요             |
|                                                  |
|   +------------------------------------------+   |
|   | 자주 나온 주제: 일, 방향성, 관계         |   |
|   +------------------------------------------+   |
|   | 반복 감정: 불안, 압박감                  |   |
|   +------------------------------------------+   |
|   | 최근의 나: 성과보다 방향을 더 고민 중    |   |
|   +------------------------------------------+   |
|                                                  |
|   [ 다음 체크인 남기기 ]                         |
+--------------------------------------------------+
```

### 메모

- 초기 구현에서는 이 통합형이 가장 단순할 수 있다
- 첫 결과와 누적 패턴을 한 화면에서 보여주면 흐름이 끊기지 않는다
- 별도 탭이나 추가 네비게이션 없이도 MVP 검증이 가능하다

## 상태별 우선순위

### P0

- 화면 1 질문 선택
- 화면 2 텍스트 입력 + 감정 태그
- 화면 3 해석 1줄

### P1

- 화면 3 데이터 부족 상태
- 화면 4 패턴 카드 3장
- 화면 5 통합형 결과/패턴 피드

### P2

- 전환 애니메이션
- 히스토리 목록
- 카드 상세 보기

## 빠른 구현 메모

- 가장 먼저 그릴 화면은 `화면 1`, `화면 2`, `화면 5`다
- 실제 MVP에서는 `화면 3`과 `화면 4`를 따로 만들지 않고 `화면 5` 하나로 합치는 방식이 효율적이다
- 텍스트 스케치 기준으로도 정보 위계는 `질문 -> 입력 -> 해석 -> 패턴` 순서를 유지해야 한다

## 추천 조합

첫 실험 버전 추천 흐름:

1. 화면 1 질문 선택
2. 화면 2 체크인 작성
3. 화면 5 통합형 결과/패턴 피드

이 조합이 가장 짧고, 구현도 단순하며, `반사 경험`을 가장 빠르게 보여준다.
