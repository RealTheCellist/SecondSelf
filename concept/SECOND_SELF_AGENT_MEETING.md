# Second Self 에이전트 회의록

## 회의 목적

`Second Self` 콘셉트를 제품 전략, 온보딩/UX, MVP 실행 관점에서 교차 검토하고, 다음 단계에서 무엇을 고정해야 하는지 정리한다.

## 참여 관점

- 제품 전략 및 포지셔닝
- 온보딩 및 첫 사용자 경험
- MVP 범위 및 실행 가능성

## 공통 결론

세 관점의 의견은 거의 같은 방향으로 수렴했다.

- 이 제품의 차별점은 `기록`이 아니라 `반사`에 있다
- 사용자는 긴 설명보다, 자기 입력이 해석되어 돌아오는 첫 결과 화면에서 가치를 느껴야 한다
- 현재 콘셉트의 가장 큰 리스크는 추상성이다
- 따라서 MVP는 더 작게 줄이고, 첫 세션의 효용을 더 구체적으로 보여줘야 한다

핵심 한 줄:

> `Second Self`의 MVP는 체크인 기능이 아니라, 첫 요약에서 "맞다" 싶은 아하 경험을 만드는 것이다.

## 논의 요약

### 1. 제품 전략 관점

- 가장 강한 부분은 `정체성 소프트웨어`라는 포지셔닝이다
- 저널링 앱과의 차별점은 입력 도구가 아니라, 패턴을 되비춰주는 거울 같은 경험이어야 한다
- 지금 상태에서는 철학은 강하지만, 첫 30초 안에 보이는 효용이 약하다

정리:

- `자기 이해`라는 표현만으로는 부족하다
- `최근 7일간 반복된 감정/주제/자기서사 한 줄 요약`처럼 결과를 더 구체적으로 약속해야 한다

### 2. 온보딩/UX 관점

- 첫 진입에서 `정체성 탐색`, `자기 서사`, `변화하는 나` 같은 표현은 매력적이지만 추상적으로 느껴질 수 있다
- 첫 세션의 목표는 깊은 분석이 아니라 `이 서비스가 나를 꽤 정확하게 비춰줄 수 있겠다`는 믿음을 만드는 것이다
- 첫 세션은 길면 안 되고, 설명보다 바로 참여하게 만들어야 한다

정리:

- 첫 화면은 제품 소개보다 `첫 질문 선택 화면`이 적합하다
- 추천 흐름은 `질문 1개 선택 -> 30초 입력 -> 즉시 해석 1줄 + 키워드 2개`다
- `반복 테마`는 충분한 데이터가 쌓이기 전에는 과장처럼 보일 수 있으므로, 첫 결과 표현을 조심해야 한다

### 3. MVP 실행 관점

- MVP-critical한 것은 `짧은 텍스트 체크인 + 1~2개의 유도 질문 + 첫 요약 화면`이다
- 초반에는 음성 입력이 꼭 필요하지 않다
- 별도 `정체성 요약 화면`은 후순위로 미루는 편이 안전하다

정리:

- 초반 화면은 `체크인 화면`과 `패턴 피드` 2개로도 충분하다
- 요약 로직은 생성형 AI를 과하게 쓰기보다 규칙형 템플릿으로 시작하는 것이 적절하다
- 최근 N개 입력에서 반복 단어, 감정 태그, 자주 나오는 문장 패턴만 집계해도 충분히 가치 검증이 가능하다

## 합의된 권고안

### 반드시 유지할 것

- `정체성 소프트웨어`라는 제품 철학
- 기록보다 해석과 반사를 전면에 두는 방향
- 커리어 전환기, 삶의 재정렬 시기 사용자에 대한 타깃 선명도

### 바로 줄여야 할 것

- 음성 입력
- 과도하게 큰 서사형 요약
- 초기부터 강한 변화 해석 문장 생성
- 화면 수가 많은 구조

### 먼저 고정해야 할 것

1. 첫 질문 선택 구조
2. 첫 세션 종료 후 보여줄 해석 1줄의 톤
3. 패턴 카드 3장의 최소 형태
4. 최근 5개 입력 기준 규칙형 요약 로직

## 회의 후 권장 MVP 재정의

### 제안 MVP

- 텍스트 전용
- 하루 1회 체크인
- 고정 질문 3개 중 1개 선택
- 자유 텍스트 입력 1개
- 감정 태그 1~2개 선택
- 최근 5개 입력 기준 패턴 카드 3장 제공

### 패턴 카드 예시

- 최근 자주 나온 주제: 일, 방향성, 관계
- 반복 감정: 불안, 압박감
- 한 줄 해석: 요즘 당신은 성과보다 앞으로의 방향을 더 자주 고민하고 있습니다
